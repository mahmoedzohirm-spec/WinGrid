# دليل مصادقة الموبايل (Phase A) ✅

## ما تم إنجازه

✅ **متجر الحالة (Zustand Auth Store)**
- `signUp()`: إنشاء حساب جديد عبر Supabase Auth + إضافة الملف الشخصي
- `signIn()`: تسجيل دخول مع جلب بيانات المستخدم
- `signOut()`: تسجيل خروج آمن
- `getCurrentSession()`: استرجاع الجلسة عند فتح التطبيق
- إدارة الأخطاء والحالة بشكل ديناميكي

✅ **شاشات المصادقة**
- `LoginScreen.tsx`: تسجيل دخول مع التحقق من الحقول
- `RegisterScreen.tsx`: تسجيل جديد مع:
  - التحقق من جودة كلمة المرور (8+ أحرف، كبيرة، صغيرة، أرقام، رموز)
  - عرض شريط قوة كلمة المرور
  - التحقق من رقم الهاتف
  - معالجة الأخطاء والتحميل

✅ **التنقل الشرطي (Conditional Navigation)**
- عرض شاشات المصادقة قبل تسجيل الدخول
- عرض شاشات التطبيق الرئيسية بعد التحقق من الجلسة
- تحميل الجلسة تلقائياً عند فتح التطبيق

✅ **شاشة الرئيسية المحدثة**
- عرض اسم المستخدم
- زر تسجيل خروج

## كيفية التشغيل

### 1. تثبيت المكتبات
```bash
cd mobile
npm install
```

### 2. إعداد المتغيرات
```bash
cp .env.example .env
```

أضف في `.env`:
```
EXPO_PUBLIC_SUPABASE_URL=your_url_here
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_anon_key_here
```

### 3. التشغيل
```bash
npm run start
```

ثم اختر:
- `i` للتشغيل على محاكي iOS
- `a` للتشغيل على محاكي Android
- أو امسح QR Code بـ Expo Go على جهازك الحقيقي

## اختبار سريع

1. سجل حساب جديد:
   - البريد: test@example.com
   - الاسم: محمد أحمد
   - الهاتف: 0512345678
   - كلمة المرور: TestPass123!

2. أغلق التطبيق وفتحه مجدداً — ستجد نفسك في الشاشة الرئيسية (الجلسة محفوظة)

3. انقر على "تسجيل الخروج" ستعود إلى شاشة تسجيل الدخول

## ملاحظات الأمان

✅ البيانات الحساسة (tokens) محفوظة بواسطة Supabase securely
✅ لم نحفظ كلمات المرور على التطبيق
✅ استخدمنا ANON key فقط (لا service_role)
✅ جميع الاتصالات عبر HTTPS
